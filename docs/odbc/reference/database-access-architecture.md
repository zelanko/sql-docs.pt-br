# <a name="database-access-architecture"></a>Arquitetura de acesso ao banco de dados
Uma das perguntas no desenvolvimento de ODBC foi qual parte da arquitetura de acesso do banco de dados para padronizar. As interfaces de programação SQL descrita na seção anterior - SQL inserido, os módulos SQL e CLIs - são apenas uma parte dessa arquitetura. Na verdade, como ODBC foi destinado principalmente para se conectar a aplicativos com base em computador pessoal minicomputador e DBMSs de mainframe, também havia um número de componentes de rede, alguns dos quais poderiam ser padronizados.
